# Design Patterns (Structural) - TypeScript examples

Este diretório contém exemplos em TypeScript de padrões estruturais com um pequeno runner para demonstrar cada exemplo.

Padrões incluídos:

- Adapter
- Bridge
- Composite
- Decorator
- Facade
- Proxy
- Flyweight

## O que são padrões estruturais

Padrões estruturais preocupam-se com a composição de classes e objetos, ou seja,
com a forma como diferentes partes de um sistema são organizadas para formar
estruturas maiores. Eles ajudam a criar sistemas mais flexíveis e fáceis de
manter, reduzindo acoplamento entre componentes e promovendo reuso.

Para que servem:

- Simplificar e organizar as relações entre objetos.
- Reduzir acoplamento entre módulos ao introduzir adaptadores, fachadas ou
  proxies.
- Compartilhar estado para economizar recursos (ex.: Flyweight).
- Permitir variações independentes entre abstrações e implementações
  (ex.: Bridge).

Cada arquivo neste diretório contém um comentário no topo explicando o padrão
específico demonstrado naquele módulo.
